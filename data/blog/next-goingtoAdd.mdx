얼마전부터 계속해서 내 블로그 만들기 프로젝트를 진행하고 있다. 중간에 컬리 해커톤에 참여하기도 했고 잠깐 몸이 안좋기도 했지만 놓지 않고 있다. 그러던 와중 지금 내가 적용하고 있는 이 블로그 프로젝트에 필요한 부분을 적어둬야 겠다는 생각이 들었다. 기록이 없으면 까먹기 쉬우니까. 지금 적어두고, 하나씩 해결해나가는 방식이 재미있을 것 같기도 하고.

## Dark 모드 적용할 때 깜빡임

다크모드를 적용할때, 화면이 깜빡이는 문제가 발생해 아직 적용하지 못하고 있다. 예상하건대, `useEffect` 등의 사용 문제인 것 같다. 또한, `nextJS`는 기본적으로 서버에서 렌더링되고 이 블로그는 모든 사이트가 이미 `html`로 사전렌더링 되어있는 상태이므로 `useEffect`가 등장하는 순서가 늦다. 일단 화면 깜빡임의 문제를 해결하고 다크모드를 적용하자.

> 해결! 링크 : [블로그에 다크모드 적용하기](http://localhost:3000/blog/next-dark-mode)

## 무한스크롤을 사용할때 문제점

모바일뷰 사용자를 위해서, 그리고 개인적인 욕심으로 전체 블로그 포스트들의 리스트를 보여주는 페이지를 무한스크롤로 작성했다. 특히 이는 모바일 뷰에서 엄청 편하긴 했다. 모바일 화면의 경우 사람들이 손가락으로 그냥 내려가면서 누르니까. 그런데 문제점은 데스크탑 뷰에서 있었다. 무한스크롤로 맨 아래까지 내려갔다가 뒤로 가기 버튼을 사용하면 현재 스크롤의 단계가 맨 위로 올라가버린다. 그래서 해당 글 근처를 보고싶다면 다시 스크롤을 내려야 한다.

정리하면, 무한스크롤을 사용하는 상황에서 뒤로가기 버튼을 눌렀을때의 스크롤바 위치 수정하기.

## 검색기능, 필터링 기능

지금 블로그에는 검색기능이나 필터링 기능이 없다. 포스트가 20여개 정도되니까 괜찮은 것 같긴 하지만 들어오는 사람이 특정 글을 찾기 쉽도록 하기위해 검색기능과 필터링 기능을 추가하도록 하자. 그런데 검색기능 같은경우는 구글에 검색하는 방식을 사용하면 생각보다 쉽게? 구현할 수 있을 것 같은데, 검색이 잘 될지 안될 지 모르겠다.

정리하면, 검색기능, 필터기능 추가하기. (우선순위를 둔다면 검색으로하자. 필터를 두기엔 `react`가 너무 많다는 느낌)

## notion API 연결

나는 블로그를 작성할때, 특별히 정리해서 공유하고 싶은 내용이 아닌 기본적인 개념과 책이나 강의를 통해 공부한 자료는 `notion` 페이지에 따로 정리하고 있었다. 그런데 얼마전 알아보니 `notion API`가 생겼고 이를 통해 `notion`에 접근 가능한 페이지를 만들 수 있을 것 같다. 물론, 이를 위한 페이지를 만드는 것도 방법이겠지만 생각보다 페이지가 너무 많아서.. 그냥 노션을 연결하는게 나을 것이라는 생각이 들었다.

정리하면, `notion API` 사용해서 `notion` 페이지만 불러오기. 얼마전 노션 멤버쉽을 업그레이드하기도 했으니 전체 공유도 가능함.
